Maintaining Digital With VCS

Version Control For Digital Projects at the Students' Union

Current Situation

  • Primary communications, sales, marketing, and transactional platform.
  • 30k+ pages across three main web assets
  • High volumes of legacy code
  • Little or no documentation
  • No access to server for file editing and/or upload
  • Site code has to be entered on the web
Oh hey, these are some notes. They'll be hidden in your presentation, but you can see them if you open the speaker notes window (hit 's' on your keyboard).

The Problem

  • Poor Brand Consistency
  • Costly to maintain
  • Slow development cycle

The Risk

Relying more on this transactional vehicle and little or no investment is being made to its ongoing maintenance and performance.

Increased costs and risk of failure if not addressed

Three Options

1 - Carry On As Normal

  • Limited History / Superficial version control
  • No external editing

2 - Manual

  • Physically download iterations of design
  • Requires suitable IT support
  • Time consuming

3 - Use a Version Control System (VCS)

What is a VCS

A system that keeps track of the changes made to a file or set of files over time that can be used to recall specific versions later.

Features of VCS

  • Backup / Restore
  • Synchronisation
  • Short/Long Term Undo
  • Track Changes
  • Track Ownership
  • Sandboxing

Off-site backups using remote repositories

Benefits

  • Save versions properly
  • Understand what happened
  • Restore previous versions
  • Collaboration without overwriting

Who Will Be Involved

  • Digital Team
  • Service Provider
  • Any others

Costs & Implementation

Upfront Costs

  • Software is free - GNU General Public License
  • Staff Time

Implementation Plan

  • Step 1 - Setup Repositories
  • Step 2 - Begin copying code to repo
  • Step 3 - Being maintenance of VCS copy
  • Step 4 - Deploy from the VCS
  • Done in parallel with current site - non-disruptive to the user
  • Service level agreement with supplier

Future Benefits

Open Source the Project

  • Students' Engaged in the development cycle
  • Actively involved in the design development and maintenance of this site
  • Potentially counting towards academic credit
  • Sharing Best Practices with other MSL clients
  • Small development team currently - slow to make changes
  • Great to have students involved in the process as a portfolio piece.
  • Half of the students unions' nationally use MSL, great opportunity to build a community

Cultural Changes

  • Changing the workflow of the team
  • Active participation in development
  • Encourage creation and maintenance of documentation
  • Education of staff in software development
  • Change to digital by default approach
  • Actively explaining why this is the correct course of action.

Conclusion

VC facilitates ensuring long term success and improvements to digital within the Union at little cost.

  • Reliable
  • Develop with confidence / Safety-net
  • Higher Quality
